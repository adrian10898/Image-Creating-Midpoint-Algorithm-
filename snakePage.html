The Beautiful Mathematics behind Nature

By Cale Retoto

For many, nature is the pinnacle of beauty. Willows whining with the wind, snakes slithering in the sand and waves washing away all that once was. Can computers even begin to replicate this fluid beauty?

In order to explore this idea I attempted to program the smooth motion of a snake. Through the design process I discovered many useful applications of mathematics and how they can be used to simulate fluid motion.

The first thought for many programmers would be to use a standard function to describe the snake's motion. However, there are many apparent limitations with this approach. A function corresponds each input or x value with a specific output or y value. For this reason an object whose movement is defined by a function is unable to turn back on itself, removing many appealing snake movements. A relation on the other hand is a collection of ordered pairs wherein each x value is not limited to one y value. For this reason an object whose movement is defined by a relation is able to preform all of the movements of a slithering snake.

In order to define the relation, parametric equations were used. Unlike normal equations these express the x and y components through separate functions in terms of t. An example of a parametric equation is shown below:

x(t) = 4t + 2
y(t) = 9t - 56

By incorporating trigonometry within these parametric equations the movement of the snake was bound to the constraints of the screen. Unlike normal functions trigonometric identities are periodic and therefore repeat themselves continuously. Although the basic graphs of sin and cos are relatively simple, combining the two together in different ratios produces very complex smooth paths. After testing multiple functions the following was chosen as appropriate path for the snake I wished to program.

To provide depth to the illustration another trigonometric function was introduced at each of the individual points. This function rendered a circular distributed set of points around each initial point thereby creating a cylinder.

This modification gave the snake the apparent depth and three dimensional nature of a tube although only two dimensions were being used. Finally colour was introduced to the program using the simple command fill(t%360) which smoothly alters the colour of the rendered points depending on the t value.

The source code of the completed program is included below and previewed at the begging of this page. Although my final product did not exactly resemble a snake, it did illustrate how simple mathematics can be used to resemble the beauty of nature.

float t = 0;
int l = 50;
int n = 20;
float r = 40;
float seperation = 0.005;
float seperationAngle = 2*PI/n;

float a(float t) {
    return 150*sin(7*PI*t);
}

float b(float t) {
    return 100*cos(5*PI*t);
}

void setup() {
    size(1000, 400);
    //strokeWeight(5);
    colorMode(HSB, 360, 100, 100);
}

void draw() {
    background(0);

    fill(255, 100);
    translate(width/2, height/2);

    for (int x=0; x<=l; x++) {
        stroke(360*t%360, 97, 100, 100);

        for (int y=0; y<=n; y++) {

            ellipse(a(t+x*seperation)+r*cos(seperationAngle*y), b(t+x*seperation)+r*sin(seperationAngle*y), 10, 10);
        }
    }

    t += 0.001;
}
